# Hyperliquid 连通性测试工具 (使用CCXT)

## 简介

这是一个使用 **CCXT** 库测试与 Hyperliquid 交易所连接状态的工具。它可以验证：

- ✅ REST API 连通性
- ✅ 认证信息有效性
- ✅ 市场数据获取能力

## 安装依赖

```bash
# 安装所有依赖
pip install -r utils/requirements_test.txt

# 或者单独安装
pip install ccxt eth-account PyYAML
```

**注意**: 
- `ccxt` 是必需的（统一交易所API库）
- `eth-account` 是可选的（用于完整的认证验证）

## 使用方法

### 基本使用

```bash
# 使用默认配置文件
python utils/test_hyperliquid_connectivity.py

# 指定配置文件路径
python utils/test_hyperliquid_connectivity.py --config config/hyperliquid_config.yaml
```

### 测试内容

工具使用CCXT库依次执行以下测试：

1. **REST API 连通性测试 (CCXT)**
   - 加载市场信息
   - 获取交易所状态
   - 获取价格数据

3. **认证信息验证**
   - 配置完整性检查
   - 私钥格式验证
   - 钱包地址格式验证
   - 私钥与地址匹配验证（如果安装了eth-account）

4. **市场数据获取测试**
   - 获取交易对价格
   - 获取订单簿数据

## 输出说明

测试结果会以颜色编码显示：

- 🟢 **绿色 (✓)**: 测试通过
- 🔴 **红色 (✗)**: 测试失败
- 🟡 **黄色 (⚠)**: 部分通过或警告
- 🔵 **蓝色 (⊘)**: 测试跳过

## 退出码

- `0`: 所有测试通过
- `1`: 部分测试通过
- `2`: 测试失败

## 故障排查

### REST API 连接失败

- 检查网络连接
- 验证CCXT是否正确安装
- 检查防火墙设置
- 确认Hyperliquid API是否可访问

### 认证验证失败

- 检查私钥和钱包地址是否已正确配置
- 验证私钥格式（应以 `0x` 开头，66个字符）
- 验证钱包地址格式（应以 `0x` 开头，42个字符）
- 确保私钥与钱包地址匹配

### 市场数据获取失败

- 检查交易所API是否正常
- 验证交易对符号是否正确
- 检查API速率限制

## 配置文件

测试工具会读取 `config/hyperliquid_config.yaml` 配置文件，使用CCXT时主要需要以下配置项：

```yaml
hyperliquid:
  api:
    timeout: 30  # CCXT会自动使用正确的API地址
  
  authentication:
    private_key: '0x...'      # 钱包私钥
    wallet_address: '0x...'   # 钱包地址（作为apiKey）
```

**注意**: CCXT会自动处理API地址，无需手动配置。

## 注意事项

⚠️ **安全提示**: 
- 配置文件包含敏感信息（私钥），请妥善保管
- 不要将包含私钥的配置文件提交到公共代码仓库
- 建议使用环境变量或加密存储来管理私钥
